1. Introduction & Project Overview

1.1 What is KrishiMitra?

KrishiMitra (meaning "Farmer's Friend" in Hindi) is a full-stack AI-powered smart farming web application designed specifically for Indian farmers. It integrates real-time weather data, AI-based disease detection, government agricultural market data, and rule-based advisory systems to help farmers make better decisions about their crops.

The platform solves these key problems:

  • What crop should I grow? — AI crop recommendation based on soil NPK, pH, weather
  • Is my crop sick? — Upload a leaf photo and get instant AI disease diagnosis
  • When should I water? — Smart irrigation scheduling with weather awareness
  • When to sell? — Live market prices + 4-week price forecasts
  • Are pests coming? — Weather-based pest and disease early warnings
  • How much will I harvest? — Yield prediction based on farm conditions
Who This Documentation Is For
This documentation is written for anyone new to full-stack web development. Every concept, file, and decision is explained from the ground up. If you understand basic HTML, CSS, and JavaScript, you can follow along and implement this entire project.

3.3 Database Setup

Option A: Using Supabase (Recommended — Free)

  1. Go to supabase.com and create a free account
  2. Click "New Project" → give it a name and password
  3. Once created, go to SQL Editor
  4. Copy the entire contents of server/models/db.sql and paste it into the SQL editor
  5. Click "Run" — this creates all the base tables (users, farms, advisories, disease_reports, market_prices)
  6. Then run the Phase 1 migration SQL to create the additional tables (crop_recommendations, irrigation_schedules, pest_alerts, yield_predictions, etc.)
  7. Go to Settings → Database → Connection String and copy the URI. Put it in your .env as DATABASE_URL

Option B: Local PostgreSQL

  1. Install PostgreSQL from postgresql.org
  2. Open pgAdmin or psql terminal
  3. Create a database: CREATE DATABASE krishimitra;
  4. Run the SQL files listed above
  5. Set DATABASE_URL=postgresql://postgres:password@localhost:5432/krishimitra

5.2 Authentication System

How JWT Authentication Works

JWT (JSON Web Token) is a stateless authentication method. Instead of storing sessions on the server, we give the user a signed token that proves their identity.

1. User sends email + password
→
2. Server verifies password
→
3. Server creates JWT with user ID
→
4. Token sent back to browser
5. Browser stores token in localStorage
→
6. Every API call includes token in header
→
7. Server verifies token → knows who's asking

Registration Flow

POST /api/auth/register
Body: { "name": "Ravi Kumar", "email": "ravi@example.com", "password": "secret123" }

Backend does:
1. Validates: name not empty, email format valid, password ≥ 6 chars
2. Checks if email already exists in database
3. Hashes password with bcrypt (10 salt rounds)
4. Inserts new user into 'users' table
5. Creates JWT token (expires in 7 days)
6. Returns: { token: "eyJ...", user: { id, name, email, role } }

Login Flow

POST /api/auth/login
Body: { "email": "ravi@example.com", "password": "secret123" }

Backend does:
1. Finds user by email in database
2. Compares password with stored hash using bcrypt
3. If match → creates JWT token
4. Returns: { token: "eyJ...", user: { id, name, email, role } }

Auth Middleware (Protecting Routes)

// Every protected route passes through this middleware:
function authMiddleware(req, res, next) {
    1. Read 'Authorization' header from request
    2. Strip 'Bearer ' prefix → extract token
    3. Verify token signature using JWT_SECRET
    4. Extract user ID from token payload
    5. Set req.user = { id, role }
    6. Call next() → allow the route handler to execute
    
    // If token is missing or invalid → return 401 Unauthorized
}

5.4 Backend Services & Business Logic

Crop Recommendation Engine

Located in services/cropRecommendation.js. Uses a scoring algorithm to match user inputs against 18 Indian crops.

Algorithm:
1. For each of the 18 crops (Rice, Wheat, Maize, Cotton, etc.):
   a. Score Nitrogen: How close is user's N to crop's ideal N range? (0-1 scale)
   b. Score Phosphorous: same approach (0-1)
   c. Score Potassium: same (0-1)
   d. Score Temperature: same (0-1)
   e. Score Humidity: same (0-1)
   f. Score pH: same (0-1)
   g. Score Rainfall: same (0-1)
   h. Bonus if soil_type matches crop's preferred soils (+1 point)

2. matchPercent = (totalScore / maxScore) × 100

3. Sort all crops by matchPercent DESC

4. Return top 8 crops with: crop name, matchPercent, compatible seasons,
   ideal conditions, soil types, expected yield per acre

Irrigation Scheduling Engine

Located in services/irrigationScheduler.js. Generates a 28-day watering plan.

Algorithm:
1. Look up crop's water needs (mm/day) for current growth stage
   - Growth stage is calculated from sowing_date:
     <30 days = seedling, <60 = vegetative, <90 = flowering, ≥90 = maturity

2. Adjust for soil retention factor:
   - Clay holds water well (factor 1.3) → needs less frequent watering
   - Sandy soil drains fast (factor 0.6) → needs more frequent

3. Adjust for irrigation efficiency:
   - Drip (90% efficient) → least water waste
   - Flood (50% efficient) → needs much more water

4. Adjust for live weather:
   - Temperature > 35°C → +30% water need
   - Humidity > 80% → -15% water need
   - Rain forecast > 20mm → -50% water need (skip irrigation)

5. Calculate interval (every 2-7 days based on effective need)

6. Generate 28 daily entries: date, water amount, "scheduled" or "skip"

Pest Alert Service

Located in services/pestAlertService.js. Contains 34 weather-based pest/disease rules.

Rule Examples:
- Rice + Humidity > 85% + Temp 25-35°C → "Rice Blast" (HIGH severity)
- Cotton + Rainfall > 30mm            → "Bollworm Risk" (CRITICAL)
- Tomato + Humidity > 75% + Temp < 20°C → "Late Blight" (HIGH)
- Any Crop + Temp > 40°C              → "Heat Stress" (CRITICAL)
- Any Crop + Rainfall > 50mm          → "Waterlogging" (HIGH)

Each rule has: condition function, severity level, pest/disease name,
              advisory message, and applicable crop(s)

Yield Predictor

Located in services/yieldPredictor.js. Estimates harvest quantity using multiplicative factors.

Algorithm:
Start with yieldFactor = 1.0, then multiply by:

1. Soil type factor: e.g., Rice on Alluvial soil = 1.1, on Sandy = 0.7
2. pH factor: ideal 5.5-7.5 = 1.0, extreme values = 0.7
3. Irrigation factor: Drip = 1.1, Sprinkler = 1.0, Rainfed = 0.65
4. Weather factor: Extreme heat = 0.7, Moderate = 0.85
5. Season factor: Right sowing season = 1.0, Wrong = 0.75

Final: predicted_yield = optimal_yield × yieldFactor × farm_area
Confidence: HIGH if yieldFactor > 0.85, else MEDIUM

Price Forecaster

Located in services/priceForecaster.js. Uses moving averages for 4-week price predictions.

Algorithm:
1. Load historical prices from database (last 60 records)
   - If < 5 records, use seasonal baseline prices (15 commodities built-in)

2. Calculate Short-term Moving Average (3-point) = recent trend
3. Calculate Long-term Moving Average (7-point) = overall trend

4. Determine trend:
   - Short MA > Long MA × 1.03 → "rising"
   - Short MA < Long MA × 0.97 → "falling"
   - Otherwise → "stable"

5. Forecast 4 weeks: extrapolate from recent change, dampened over time

6. Sell/Hold recommendation:
   - Falling + current > forecast avg → "Sell now"
   - Rising → "Hold for better prices"
   - Stable → compare to short-term average

Auth Protection

There is no <PrivateRoute> wrapper component. Instead, authentication is enforced via the Axios response interceptor: if any API call returns a 401 Unauthorized response, the interceptor automatically clears localStorage and redirects to /login. Individual pages also check for a token on mount and redirect if missing.

Axios Instance Configuration

// api/axiosInstance.js
const api = axios.create({
    baseURL: import.meta.env.VITE_API_URL || 'http://localhost:5000'
});

// Request interceptor: attach JWT to every request
api.interceptors.request.use(config => {
    const token = localStorage.getItem('token');
    if (token) config.headers.Authorization = `Bearer ${token}`;
    return config;
});

// Response interceptor: handle 401
api.interceptors.response.use(
    response => response,
    error => {
        if (error.response?.status === 401) {
            localStorage.removeItem('token');
            localStorage.removeItem('user');
            window.location.href = '/login';
        }
        return Promise.reject(error);
    }
);

6.3 Page-by-Page Breakdown

Login & Register Pages

Split-screen layout: left 55% is a gradient hero panel with branding and feature highlights; right 45% is the form. On mobile, the hero panel hides and only the form is shown.

Dashboard

The main landing page after login. Shows a greeting (Good morning/afternoon/evening + user name), a 3-column grid of 11 feature cards (each links to its respective page), a Government Schemes widget, and a Seasonal Farming Tips section.

Farms (CRUD)

Expandable "Add New Farm" form at the top. Below it, a grid of farm cards showing name, crop, area, soil, pH, irrigation, sowing date, and coordinates. Each card has a delete button. Form fields: farm name, area (acres), soil type (dropdown), current crop (dropdown), soil pH, irrigation type, sowing date, previous crop, latitude/longitude, district, state.

Crop Advisory (SimplifiedAdvisory)

Two sections: (1) Input form with NPK sliders/inputs, pH, and weather data (temperature, humidity, rainfall). Has an "Auto-fill" button that fetches live weather. (2) Results panel showing a Soil Health Score (Good/Fair/Poor based on NPK ranges), the recommended crop, and fertilizer advice. Includes tabs for "Latest Result" and "History".

Crop Recommendation

Two tabs: "Manual Input" (enter N/P/K/temp/humidity/pH/rainfall manually) and "From My Farm" (select a registered farm to auto-populate). Results show top 8 matching crops with match percentage, compatible seasons, and expected yield.

Disease Detection

Drag-and-drop image upload area for leaf photos. Supports JPG, PNG, WEBP up to 5MB. After analysis, shows detected disease name, confidence score (with color-coded badge), and recommended treatment. Below the main area is a "How Disease Detection Works" section explaining the 3-step process. Links to Disease History page.

Irrigation Scheduler

Farm dropdown selector. Generates a 28-day schedule showing: Schedule Summary (growth stage, water need/day, interval, estimated harvest date), Weather Advisory with current conditions, and a day-by-day schedule table with water amounts and skip/scheduled status.

Pest Alerts

Farm multi-select (or "All My Farms"). Shows severity-color-coded alert cards with pest/disease name, type (Fungal/Bacterial/Viral/Pest/Abiotic), advisory message, and weather snapshot used for the evaluation.

Yield Prediction

Farm dropdown. Shows predicted yield in kg, yield per acre, percentage of optimal, confidence level, impact factors breakdown (soil, pH, irrigation, weather, season), harvest window dates, and a visual comparison bar (Minimum ↔ Your Prediction ↔ Optimal).

Market Prices

Search interface with state, district, and commodity dropdowns. Shows results in a card grid with min/max/modal prices per quintal. Summary stats at top (best price, price range, markets found). Agmarknet data source with fallback notice.

Price Forecast

Commodity + State selectors. Shows current price, trend direction (rising/falling/stable), short & long-term averages, a trend chart (historical + 4-week forecast), Sell/Hold recommendation with explanation, and weekly forecast breakdown.

Farm Map

Interactive Leaflet.js map showing all farms with lat/lon as markers. Sidebar listing farms. Click a farm to see popup with details. Farms without coordinates are listed separately with a reminder to add them.

Reports & Export

Four export sections: Farm Details, Crop Advisories, Disease Detections, Yield Predictions. Each has CSV and PDF download buttons. Plus a "Complete Farm Report" button that generates a comprehensive PDF with all data.

Soil Lab Locator

Two search modes: GPS-based (uses device location + Haversine distance) and State/District selection. Results show lab name, type (Govt/Private), distance, address, phone, fee, turnaround time, and services. Includes a "How to Collect Sample" guide and Google Maps links.
